Overview

Bridging environmental perception and wireless communication.

V2X-SenseComm is designed to support learning-based research at the intersection of vehicular sensing, ray-tracing communication, and beam management. The platform connects sensing observations from dynamic V2X environments with communication channels and beam-domain labels.

The current release focuses on radar-aided near-field beam prediction for XL-MIMO V2I communications. It aims to provide a reproducible dataset and benchmark for studying how sensing information can reduce beam training overhead and improve beam management in high-mobility vehicular scenarios.

Multimodal Sensing

Radar observations are provided in the initial release, with extensible interfaces for future vision, LiDAR, positioning, and map-based modalities.

Ray-Tracing Channels

Communication channels are generated from geometry-aware V2X scenarios with infrastructure nodes, target vehicles, and dynamic blockers.

Beam-Domain Labels

Beam indices, beam responses, or beam heatmaps are generated from predefined XL-MIMO codebooks for learning-based beam prediction.

Expandable Platform

The website and repository are organized for future datasets, scenarios, baselines, papers, and open-source releases.

Scenarios

Representative V2X environments with sensing and communication coupling.

V2X-SenseComm currently focuses on infrastructure-assisted V2I communication scenarios, where a roadside base station or sensing node observes the surrounding traffic environment and supports beam prediction for target vehicles.

The scenario design emphasizes the coupling between environmental geometry, radar sensing observations, propagation paths, and beam-domain communication labels. Future versions will include more diverse road layouts, blockage patterns, and mobility configurations.

Scenario elements

  • Roadside infrastructure nodes / base stations
  • Target vehicles acting as communication users
  • Obstacle vehicles causing dynamic blockage
  • Urban roads, street canyons, and NLoS layouts
  • Radar sensing observations and ray-tracing communication channels

Dataset

Organized for reproducible sensing-assisted communication research.

V2X-SenseComm/
├── scenarios/              # Scenario descriptions, metadata, and layouts
├── sensing/
│   ├── radar/              # Radar observations, spectra, or radar tensors
│   ├── camera/             # Reserved for future RGB images
│   └── lidar/              # Reserved for future point clouds
├── communication/
│   ├── channel/            # Ray-tracing channel responses
│   ├── beam_power/         # Beam-domain received power or beam responses
│   └── codebook/           # XL-MIMO beam codebooks
├── labels/
│   ├── best_beam_index.csv # Optimal beam labels
│   └── beam_heatmap/       # Optional beam map or heatmap labels
├── splits/                 # Train / validation / test splits
├── configs/                # Scenario, radar, channel, and codebook configs
├── scripts/                # Data generation and preprocessing scripts
└── README.md

What is included?

The dataset is designed to provide aligned sensing observations and communication labels for learning-based beam prediction.

  • Scenario metadata and vehicle states
  • Radar sensing observations or angular spectra
  • Ray-tracing-based communication channels
  • Beam-domain labels for XL-MIMO V2I links
  • Benchmark-ready train, validation, and test splits

Data Format

Each sample links sensing observations, scenario metadata, and communication beam labels through a consistent sample index.

Data Loading

Example Python dataloaders will be provided for reading radar inputs, beam labels, and train/test splits.

Visualization

Visualization scripts will support scenario rendering, radar spectrum visualization, and beam heatmap display.
